# Documentation Guide

This folder contains the core customer-facing and delivery-facing documentation for the Supplier Intelligence Dashboard.

Use these documents as a complete package during stakeholder alignment, user onboarding, and implementation.

## What Is Included

- `product-overview.md`  
  Business-facing one-pager for leaders and stakeholders. Use this to explain product value, outcomes, and scope.

- `onboarding-guide.md`  
  Step-by-step guide for procurement users. Use this for first-time training and day-to-day adoption support.

- `implementation-runbook.md`  
  Delivery playbook for implementation teams. Use this to structure discovery, data mapping, go-live, and hypercare.

## Recommended Reading Order

1. `product-overview.md` (scope)
2. `implementation-runbook.md` (plan deployment approach)
3. `onboarding-guide.md` (prepare user enablement)

## How to Use This Documentation

### For Procurement Leadership

- Start with `product-overview.md` to confirm strategic fit and expected outcomes.
- Use key outcomes and scope sections during internal buy-in conversations.

### For Delivery and Customer Success Teams

- Follow `implementation-runbook.md` phase-by-phase.
- Use the checklists to track readiness and reduce go-live risk.
- Reuse discovery questions and workshop guidance in customer sessions.

### For End Users

- Use `onboarding-guide.md` as the baseline training path.
- Reference specific sections (weights, exceptions, notifications) during live walkthroughs.

## Maintenance Guidance

- Keep terminology consistent across all files (supplier visibility, supply continuity, corrective actions, quota allocation).
- Update all related documents when workflow changes are introduced in the product.
- Record major process updates in the runbook first, then reflect user-facing changes in the onboarding guide.

## Ownership

Recommended ownership model:

- Product messaging and scope: Product / Customer Success
- Runbook process quality: Implementation / Delivery Lead
- User instructions: Customer Success / Enablement

Consistent ownership keeps documentation accurate and improves time-to-value for new customer teams.
